How it works

Objects/Agents

When modelling your system of interest with Tree of Knowledge, you are not really making a new model, rather you are using a part of a holistic model of the world and, if necessary, adding to it.
At the core of this holistic model lies a hierachy of objects (Figure 1). Each object in this hierachy has a set of own or inherited behaviours, that specify how it's internal state changes depending on inside and outside conditions (agent-based, monte-carlo simulation). To create a simulation model of a system you use these objects with their pre-defined behaviours and, if necessary, add new behaviours to them model.

Learning Behaviours

Your simulation models are automatically verified with data from the knowledge base - the simulation results must match the real observations of the system as accurately as possible.
Behaviour rules are also learned this way: for each simulation in which a certain object occurs, the system automatically checks which combination of the object's rules produce the most correct simulation. Rules that improve the results of many different simulations are considered to be more correct than rules that worsen their results.
All this is done within the rigorous formalism of likelyhood-free bayesian inference => the learned probability distributions accurately reflect the certainty about the rules.

Knowledge Base

Above we mentioned a centralized repository of object behaviours. The second key component of the Tree of Knowledge's holistic model is the knowledge base.
A knowledge base is a database in which the data is automatically integrated. Like with existing knowledge bases (dbpedia, yago, etc.), any structured data can be uploaded to the Tree of Knowledge's knowledge base. The Tree of Knowledge's knowledge base is however much more advanced. For any datapoint in the knowledge base we know everything: when it was observed, what attribute/relation of what specific object was being observed, etc. Furthermore, the data quality is assured through strict data monitoring and data validation.
This powerful knowledge base enables much of the other functionality such as automatic model validation, automatic model initalisation and advanced querying.

Growing the world's most detailed and accurate model

Both the repository of object behaviours and the knowledge base grow over time.
As users upload datasets to the knowledge base, it grows to an extensive data portal, accurately cataloging large parts of human knowledge.

The growth/refinement of object behaviours is even more impressive! As users make new models portraying the objects in new scenarios (the scenarios they are trying to understand/for which they have data), the object's behaviour gets increasingly comprehensive and exact (due to bayesian updating). If objects have been simulated and evaluated in sufficiently many environments, they can be used to create the by far the most accurate models of the world!

Accurate predictions

The increasing exactness of behaviour rules can be observed in the narrower proability density functions associated with them.
In other words: the uncertainty about behaviour rules (due to imperfect knowledge of the world) are always accurately determined through likelyhood-free bayesian inference and bayesian updating.
When making predictions these uncertainties are accurately propagated through the system through monte-carlo simulations.
