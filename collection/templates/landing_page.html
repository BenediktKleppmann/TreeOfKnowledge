Don't model an individual dataset,
model the world!
All existing Statistics- and Machine Learning- models are trained on an individual dataset. They don't learn general patterns, but only those occuring in this dataset. The Tree of Knowledge models are however trained on many different datasets with observations from many different scenarios. The findings from Tree of Knowledge are therefore generally valid and many orders of magnitude more accurate and robust.
Don't just make predictions,
fully understand the system!
At the core of Tree of Knowledge lies a Probabilistic Programming engine. This allows the user to easily understand and model complex behaviours, even if they are only indirectly observed.
Be certain about uncertainty!
Due to bayesian inference and monte-carlo simulations, the uncertainties of the system are accurately captured and modelled. The error-margins of the simulations exactly reflect the true uncertainty of the system.
Be part of something great!
By modelling your dataset in your specific domain you are automatically building uppon and contributing to a powerful holistic model of the world. This model will one day be able to accurately simulate all interactions in an economy.


How it works


Hierachy of Objects

At the core of the holistic model lies a hierachy of objects - see figure 1. The objects in this hierachy have a set inherited or own of behaviours. To model/simulate a system you use these objects and, if necessary, add additional specific behaviours to them. The simulations are automatically verified with data from the knowledge base - the simulation results must match the real observations of the system.

Learning Behaviours

Behaviour rules are learned the same way. For each simulation in which a certain object occurs, the system automatically checks which combination of the object's rules produce the most correct simulations. Rules that improve the results of many different simulations are considered to be more correct than rules that worsen the results. Key Components


Integrated data repository

Data from any source that is uploaded to this online platform and is mapped to the holistic data model. The holistic data model contains all the data and meta-data to exactly specify any data value.

Simulation models

For modelling real-world processes, simulation models are more accurate and insightful than all models from machine learning, statistics, traditional AI and other sciences. Especially when paired with the automatic model evaluation.

Automated model evaluation

As the simulations use the same objects, attributes and scenario descriptions (meta-data) as the integrated data repository. This enables the automatic cross-validation and scoring of models.

Collaborative model building

The automated model evaluation allows for collaborative model building. People can work together to build the most accurate models.

Compounding Simulations

Model building is facilitated not only by the possibility to modify existing models, there platform also automatically suggests behavior rules for the objects in your simulation.
The different behavior-rules from objects are collected from across different simulations. The accuracy of these behaviours are automatically evaluated with the data from all the real-world systems that contain these objects.

Powerful analysis environment

The analysis environment allows users to
  • ask What-if questions by modifying initial conditions
  • optimize simulation parameters to e.g. maximize KPIs
  • gain an intuitive understanding through the powerful visualization functionality
